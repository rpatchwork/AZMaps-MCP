## Lessons Captured

### 1. Container Apps Revision Model
- Each deployment creates new revision with auto-generated suffix
- Old revisions kept for rollback (configurable retention)
- Traffic splitting available for blue/green deployments

### 2. Health Probe Configuration
- Liveness probe prevents zombie containers
- Initial delay (5s) allows startup time
- Path must match server implementation (`/healthz` not `/health`)

### 3. ACR Integration
- Managed Identity simplifies authentication (no registry passwords)
- AcrPull role sufficient for image pull operations
- Image updates trigger automatic redeployment

### 4. Deployment Speed
- ~2 minutes for Container Apps to pull new image and start
- Fast iteration cycle for HTTP transport changes (~10 minutes end-to-end)
