## Analysis Phase — Root Cause

**Problem Identified:**
```typescript
// src/server.ts (line 245) — BEFORE
const transport = new StdioServerTransport();
await server.connect(transport);
console.log('[SERVER] MCP Server running on stdio transport');
```

**Why This Failed:**
- `StdioServerTransport` reads JSON-RPC from STDIN, writes to STDOUT
- Never opens HTTP port or listens for network connections
- Container Apps ingress configured for HTTP on port 3000
- Result: Health check `GET /health` times out (no HTTP server to respond)

**From MCP Best Practices:**
> ❌ DON'T USE: StdioServerTransport (for local CLI MCP servers)  
> ✅ USE: SSEServerTransport (HTTP with Server-Sent Events) for Azure Container Apps

**Complexity Assessment:** LOW-MEDIUM
- MCP SDK provides `SSEServerTransport` natively
- Express.js is standard Node.js HTTP framework
- All tool implementations unchanged (transport layer only)
- ~70 lines of code to modify

### 2. src/server.ts
```typescript
// NEW IMPORTS
import express from 'express';
import { SSEServerTransport } from '@modelcontextprotocol/sdk/server/sse.js';

// HTTP SERVER SETUP
const app = express();
app.use(express.json());

// HEALTH CHECK ENDPOINT
app.get('/healthz', (_req, res) => {
  res.status(200).json({ 
    status: 'healthy', 
    timestamp: new Date().toISOString() 
  });
});

// MCP JSON-RPC ENDPOINT
app.post('/message', async (req, res) => {
  try {
    const transport = new SSEServerTransport('/message', res);
    await server.connect(transport);
  } catch (error) {
    console.error('[ERROR] MCP transport connection failed:', error);
    res.status(500).json({ 
      error: 'Internal server error', 
      message: error instanceof Error ? error.message : String(error) 
    });
  }
});

// START HTTP SERVER
const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
  console.log(`[SERVER] MCP Server running on HTTP transport (port ${PORT})`);
  console.log(`[SERVER] Health check: GET /healthz`);
  console.log(`[SERVER] MCP endpoint: POST /message`);
});
```

**Changes Made:**
- Removed: `StdioServerTransport()` initialization
- Added: Express HTTP server on port 3000
- Added: `/healthz` health check endpoint
- Added: `/message` MCP JSON-RPC endpoint with SSE
- Added: Error handling for transport connection failures

## Lessons Captured

### 1. Transport Layer Selection is Core Architecture
- Not a feature/polish trade-off — fundamental communication layer
- Deployment model (Container Apps HTTP ingress) dictates transport type
- StdioServerTransport: CLI tools only
- SSEServerTransport: Network-accessible services

### 2. MCP SDK Patterns
- SSEServerTransport is standard for Azure hosting
- Express integration is straightforward (~30 lines)
- Error handling should wrap transport.connect() calls

### 3. Health Endpoint Design
- Separate health check from MCP endpoint (`/healthz` vs `/message`)
- Return timestamp for probe debugging
- Use 200 OK for healthy, 500 for unhealthy

### 4. Code Change Impact
- Transport layer change: Core architecture
- Tool implementations: Zero changes required
- Risk profile: Medium (core component), but mitigated by standard patterns
